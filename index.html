Frequently Asked Question

The difference between var, let, and const
In JavaScript, var is the old way to declare variables — it’s tied to functions, can be redeclared and reassigned, and gets hoisted to the top of its scope. let is newer, works only within the block it’s in, can change its value but can’t be redeclared in the same block. const is for constants — it can’t be reassigned or redeclared, though if it’s an object or array, we can still change what’s inside it. A good rule is to use const by default, let when you need to change values, and skip var unless you really need it.
The difference between map(), forEach(), and filter()
In JavaScript, forEach() loops through an array to perform actions but returns nothing, map() transforms each element and returns a new array, and filter() selects elements that meet a condition and returns a new array.
Explain arrow functions and how they are different from regular functions
Arrow functions are a shorter way to write functions using =>. They inherit this from the surrounding code, can’t be used with new, and don’t have their own arguments. They’re perfect for concise, modern JavaScript.
How JavaScript Promises work
JavaScript Promises handle asynchronous operations by representing a value that may be available now, later, or never. Use .then() for success and .catch() for errors
How closures work in JavaScript
a closure happens when a function keeps access to variables from the place where it was created, even after that outer function is done running. It’s like the function carries its surrounding environment with it, letting you use or remember values later.
